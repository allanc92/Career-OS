# Notes

This folder is freeform space for human-authored notes: research, ideas,
questions, contact notes, interview notes, career reflections, or anything else
that may help later.

Use a descriptive filename such as `YYYY-MM-DD-topic.md`. A useful optional
shape is:

```markdown
# [Topic]

- Date: [YYYY-MM-DD]
- Related opportunity: [OPP-ID or None]
- Source: [Where this came from]

## Notes

[The user's own notes]

## Follow-up

- [Action, owner, and due date]
```

Structure is optional. Preserve the user's wording, uncertainty, and
attribution. An assistant may organize or summarize only content the user
provides, and must show its proposed version before saving unless the user
directly asks to save that content.

Real notes may contain personal or sensitive information and should remain
untracked. Put employer-confidential or especially sensitive background in
`Private/Background/` instead. The first real save requires the one-time privacy
confirmation.
