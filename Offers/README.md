# Offers

Use this folder to compare and evaluate written offers. Copy
`_TEMPLATE_Offer_Analysis.md` to `[Organization]_Offer_Analysis.md` for each
offer. Real offer files are confidential and should remain untracked.

## Offer statuses

Use exactly one:

- `Received`: written terms arrived
- `Reviewing`: terms and fit are being evaluated
- `Clarifying`: unanswered terms are with the organization
- `Negotiating`: changes are being discussed
- `Accepted`: user accepted
- `Declined`: user declined
- `Expired`: response window ended
- `Withdrawn`: organization withdrew the offer

Map the current offer to the pipeline as follows:

- `Received`, `Reviewing`, or `Clarifying` -> `Offer`
- `Negotiating` -> `Negotiating`
- `Accepted` -> `Accepted`
- `Declined` -> `Withdrawn`
- `Expired` or offer `Withdrawn` -> `Closed`

## Accuracy and approval

Copy terms exactly from the written offer and label all calculations,
assumptions, estimates, currencies, and tax exclusions. Use
`[TBD]` for missing terms. Career OS can organize decisions but
does not replace legal, tax, immigration, or financial advice.

Show the complete analysis before creating it. Show any proposed negotiation
language and pipeline update separately. Nothing is sent to an organization,
and no status changes to `Accepted` or `Declined`, without the user's explicit
approval. A request to analyze an offer does not authorize communication or
acceptance.
