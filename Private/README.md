# Private

This folder is for confidential, local-only content. It is not a publishing
area and must remain excluded from version control.

Use `Private/Background/` for confidential employer, client, project, legal, or
other background material that may help private reasoning. Create that
subfolder locally when needed. `.gitignore` may ignore the entire subfolder, so
a placeholder file might not be tracked. That is expected.

## Rules

- Never copy Private content into a public file, application, resume, message,
  or example.
- An assistant may use it only when the user requests work that needs it.
- Before using any detail outside `Private/`, show the exact sanitized wording,
  cite the private source locally, and wait for explicit approval.
- If safe use is uncertain, summarize the constraint without exposing the
  confidential detail and ask the user.
- Never commit or push this folder. If a backup is needed, use a storage option
  the user has explicitly chosen for confidential material.

Before the first real save anywhere in Career OS, ask:

> This is your private copy. Real details stay on your computer and out of any
> public repository. Your AI provider may process content you share with it.
> OK to save?

This folder protects against accidental repository publication only. It does
not change the privacy terms of the AI provider or storage service the user
chooses.
