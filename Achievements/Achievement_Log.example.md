# Achievement Log

This tracked file is the schema only. Onboarding creates the real, untracked
`Achievement_Log.md`.

## Entry template

### [ACH-YYYYMMDD-NNN] [Outcome-focused title]

- **Date:** YYYY-MM-DD or YYYY-MM; use `[TBD]` if unknown
- **Status:** [Draft | Grounded | Archived]
- **Context:** [What was happening and why it mattered]
- **Contribution:** [What you personally decided, built, changed, or led]
- **Outcome:** [What changed and who benefited]
- **Metrics:** [Confirmed measure and comparison, or None confirmed]
- **Sources:** [Resume section, note path, artifact, or User confirmed YYYY-MM-DD]
- **Skills:** [Skill 1, Skill 2]
- **Tags:** [lowercase-tag, reusable-tag]
- **Story link:** [STORY-NNN | Candidate | None]
- **Sensitivity:** [Shareable | Internal | Confidential]
- **Added:** [YYYY-MM-DD]
- **Updates:** [YYYY-MM-DD: correction or added evidence, or None]

## Neutral example

### [ACH-20260115-001] Reduced weekly reporting effort

- **Date:** 2026-01
- **Status:** Grounded
- **Context:** A team assembled the same operational summary by hand each week.
- **Contribution:** Mapped the repeated steps, created a reusable workflow, and
  documented ownership.
- **Outcome:** The team spent less time preparing the summary and used a
  consistent set of inputs.
- **Metrics:** Reduced preparation time from 4 hours to 90 minutes per week.
- **Sources:** User confirmed 2026-01-15
- **Skills:** Process improvement, stakeholder alignment, documentation
- **Tags:** efficiency, operations, cross-functional
- **Story link:** Candidate
- **Sensitivity:** Shareable
- **Added:** 2026-01-15
- **Updates:** None
